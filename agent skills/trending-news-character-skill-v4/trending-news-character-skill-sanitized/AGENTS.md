# Project Context

## What this project is
This directory is the operating system for the configured character content engine.

Its purpose is to:
- monitor relevant niche narratives, source material, and fast-moving story signals
- identify the best narratives to cover
- turn them into viral short-form content
- support posting and scaling the configured character accounts

This project is not generic research or generic media work.
Its core job is short-form viral content creation and account growth.

## Primary job in this project
When working in this project, default to this mission:
- find the best narrative candidates
- prioritize narratives that are timely, useful, emotionally strong, and financially relevant
- convert those narratives into short-form video content
- support the production of 3 short-form videos per day
- optimize for growth, clarity, usefulness, and repeatable account scaling

Unless the user explicitly redirects you, assume this is the core function of the system.

## Core project rules
- The main purpose is creating and scaling short-form content.
- Do not drift into generic analysis without converting it into usable content outputs.
- Narrative selection should prioritize:
  - virality potential
  - strategic importance
  - practical relevance
  - usefulness to the configured audience
  - clarity of narrative
- The system should continuously think in terms of what becomes a strong short-form video.

## Output expectation
The default daily goal is:
- 3 short-form videos per day
- designed for:
  - Instagram
  - X
  - Threads
  - Facebook
  - YouTube Shorts
  - TikTok

Each narrative should be translated into content that explains:
- what happened
- why it matters
- who benefits
- how it relates to attention, behavior, risk, opportunity, or practical consequence

## Research and topic selection rules
- Use `config/research.json` to determine mode.
- If `manual_topic`, skip discovery and verify the supplied topic.
- If `x_keywords` or `x_list`, use configured sources and choose the strongest real narrative.
- Prefer narratives with strong audience, technological, cultural, practical, or financial consequences.
- Prefer narratives that can be turned into hard, useful, emotionally strong content quickly.

## Production rules
- Do not improvise around missing config values.
- Do not invent provider IDs, secrets, or required fields.
- Keep approved script meaning intact through production.
- Do not truncate persistent titles.
- Return clear production artifacts and final output paths.

## Brand alignment
- The voice and worldview come from `SYSTEM.md` plus the brand docs in this folder.
- This project exists to operationalize that mission into repeatable content output.
- If there is tension between generic repo behavior and the content mission, prioritize the content mission while inside this project.

## Success standard
A good result in this project is not just “correct analysis.”
A good result is:
- high-quality short-form content
- useful and clear framing
- strong hooks
- useful audience-relevant insight
- repeatable audience growth
- forward motion on scaling the configured character brand
