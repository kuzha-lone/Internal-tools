# Story Evaluation

Use this file inside `viral-research-skill` when deciding whether a cluster deserves to move to scripting.

## First Filter: Is It A Story?

A cluster is a usable narrative only if it has:

- a clear event, change, or debate
- repeated supporting posts
- a consequence that can be explained
- a clean sentence-level summary

If you cannot summarize it in one sentence, it is not ready.

## Consequence Test

Ask:

1. Does this affect price, liquidity, volume, wallet behavior, or capital rotation?
2. Does it change what the audience should pay attention to today?
3. Does it create a warning, setup, or asymmetric opportunity?

If the answer is no across all three, reject it.

## Mechanism Test

Identify the underlying mechanism:

- automation pressure
- regulatory shift
- capital rotation
- platform dependency
- distribution change
- price signal
- narrative manipulation
- infrastructure bottleneck

If there is no mechanism, the angle is too weak.

## Working-Class Relevance Test

Translate the narrative into one of these stakes:

- job pressure
- income opportunity
- cost pressure
- capital preservation
- timing advantage
- technology adoption risk
- control or surveillance risk

If the stakes do not map cleanly, the narrative is not strong enough.

## Advice Potential Test

Ask:

- can this end with a real next move?
- does the audience still control something meaningful here?

If not, the narrative may still be interesting, but it is not a strong fit for this character workflow.

## Rejection Triggers

Reject the narrative if it is:

- only engagement with no consequence
- only outrage with no mechanism
- only hype with no edge
- only abstraction with no actionability
- too redundant with a stronger candidate
